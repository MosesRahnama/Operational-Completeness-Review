# Extracts Folder

Purpose: Short, citable excerpts with metadata explaining why each item matters.
Contents: Folder overview and links to extract categories.
Contents: One file per reviewed artifact, each starting with a metadata header (purpose, source path, hash, failure explanation) followed by an excerpt.

Subfolders
- `01_transcripts/` — model transcripts and chat logs (Category 1).
- `02_tests/` — strict test runs and operator protocols (Category 2).
- `03_failed_attempts/` — failed proof attempts and diagnostics (Category 3).
- `04_analysis/` — cross‑file analyses and synthesis notes (Category 4).
- `05_misc/` — miscellaneous/reference/planning items (Categories 4–5).
